Einführung in Static Analyzer

Verwenden Sie die statische Analyse, um Quellcode auf Sicherheitslücken zu überprüfen. Zu diesem Zweck muss ein kleines Dienstprogramm heruntergeladen werden. Wenn Sie das Dienstprogramm auf Ihre lokale Festplatte extrahieren, können Sie über seine Befehlszeilenschnittstelle (CLI) Sicherheitsanalysen für alle unterstützten Sprachen durchführen. Sie können aber auch ein integriertes Installationsprogramm ausführen, mit dem Plug-ins für die statische Analyse zu Eclipse, Maven, IntelliJ IDEA oder Visual Studio hinzugefügt werden. In Eclipse und IntelliJ IDEA können Sie Java-Projekte scannen und in Visual Studio können .NET-Lösungen (C#, ASP.NET, VB.NET) gescannt werden.

Vorgehensweise

  • Informationen zur Verwendung des Application Security on Cloud-Service in IBM Cloud Marketplace finden Sie unter http://www.ibm.com/support/knowledgecenter/SSYJJF_1.0.0/ApplicationSecurityonCloud/Welcome.html.
  • Informationen zur Verwendung von Application Security on Cloud for Bluemix finden Sie unter https://console.ng.bluemix.net/docs/services/ApplicationSecurityonCloud/index.html.
  • In den übrigen Teilen dieser Readme-Datei sind die Themen 'Installation' und 'Verwendung' von Static Analyzer Client Utility abgedeckt.

Static Analyzer Client Utility einrichten

Für die statische Analyse ist ein kleines Clientdienstprogramm verfügbar, das Sie herunterladen können. Wenn Sie das Dienstprogramm auf Ihre lokale Festplatte extrahieren, können Sie über seine Befehlszeilenschnittstelle (CLI) Sicherheitsanalysen durchführen. Sie können aber auch ein integriertes Installationsprogramm ausführen, mit dem das Plug-in für die statische Analyse zu den Maven-Buildumgebungen oder den integrierten Entwicklungsumgebungen (IDEs) von Eclipse, IntelliJ IDEA oder Visual Studio hinzugefügt wird.

Informationen zu diesem Vorgang

Bei der Verwendung einer statischen Analyse scannen Sie Quellcode, um eine Datei zu generieren, die in die Cloud hochgeladen wird. Die Datei, eine verschlüsselte IRX-Datei (.irx) wird mit der Traceanalyse gescannt, um Sicherheitslücken aufzuspüren. Wenn die IRX-Datei für Compilersprachen generiert wird, werden Anwendungsbytecodedateien in eine Intermediate Representation des Codes konvertiert. Wenn eine IRX-Datei für Scriptsprachen generiert wird, werden Quellendateien in die verschlüsselte IRX-Datei integriert.

Wenn Sie die Plug-ins für Eclipse, IntelliJ IDEA oder Visual Studio verwenden, erfolgt das Generieren und Hochladen der IRX-Datei von der integrierten Entwicklungsumgebung (IDE) automatisch, wenn Sie Aktionen für die statische Analyse verwenden.

Tipp: Wenn Sie nicht beabsichtigen, die statische Analyse in einer speziellen integrierten Entwicklungsumgebung (IDE) auszuführen, aber die betreffende integrierte Entwicklungsumgebung bereits installiert ist und Sie davon ausgehen, dass Sie sie künftig verwenden werden, kann es beim Ausführen des Installationsprogramms ratsam sein, das Plug-in auf dieser integrierten Entwicklungsumgebung zu installieren, sodass Sie das Installationsprogramm nicht zu einem späteren Zeitpunkt erneut ausführen müssen.
Direktaufruf: Wenn die Eingabe von yes bzw. no im Installationsprogramm erforderlich ist, reicht die Verwendung der Abkürzungen y bzw. n. Wenn Sie Abkürzungen in einem Installationsprogramm eingeben, das auf einer Maschine mit einer anderen als der englischen Ländereinstellung ausgeführt wird, so müssen Sie y bzw. n als Abkürzungen eingeben.
Wichtig: Die Client Utility darf nicht in ein Verzeichnis platziert werden, das Nicht-ASCII-Zeichen enthält.

Vorgehensweise

Gehen Sie wie folgt vor, um Client Utility einzurichten:

  1. Wählen Sie die Plattform aus, auf der Client Utility ausgeführt werden soll, und klicken Sie auf Herunterladen. Daraufhin wird die Datei SAClientUtil_<Version>_<Betriebssystem>.zip heruntergeladen (wobei <version> die aktuelle Version von Client Utility und <os> das Betriebssystem ist, auf dem Client Utility verwendet werden soll).
    Anmerkung: Sie können das Willkommensformular erneut öffnen, indem Sie die Option zur Vorgehensweise für die Erstellung einer IRX-Datei in dem Formular auswählen, mit dem Sie eine IRX-Datei auswählen können.
  2. Extrahieren Sie die Datei in ein lokales Verzeichnis.
  3. Gehen Sie wie folgt vor, wenn Sie beabsichtigen, die Befehlszeilenschnittstelle für die IRX-Dateigenerierung, zum Hochladen von IRX-Dateien oder zum Verwalten von Scans zu verwenden: Fügen Sie die Position des Verzeichnisses \bin der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip zu Ihrer Umgebungsvariablen PATH hinzu. Wenn Sie dies nicht tun, müssen Sie bei allen Befehlen bei der Eingabe jeweils das Verzeichnis \bin der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zipangeben.
    Wichtig: Wenn Sie beabsichtigen, .NET mit der Befehlszeilenschnittstelle (CLI) oder mit Microsoft Visual Studio zu scannen, müssen die folgenden drei .bat-Dateien (in der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip) mit Administratorrechten ausgeführt werden:
    • bin\register.bat
    • walalib\dotnet.libs40\RegisterInGAC.bat
    • walalib\dotnet.libs35\RegisterInGAC.bat

    Diese Dateien können manuell ausgeführt werden. Alternativ können Sie auch die Datei install-plugins.bat als Administrator ausführen. (Falls Sie nur die Befehlszeilenschnittstelle verwenden, führen Sie die Datei install-plugins.bat aus, geben Sie jedoch no ein, wenn Sie gefragt werden, ob das Plug-in für die statische Analyse installiert werden soll.)

    Wenn Sie diese Dateien nicht als Administrator ausführen, wird in der Konsole immer dann eine Warnnachricht angezeigt, wenn die CLI verwendet wird. Diese Nachricht kann ignoriert werden, wenn Sie Java-Ziele scannen.

    Tipp: Geben Sie nach dem Ändern der Variablen PATH den Befehl appscan version (Windows) oder den Befehl appscan.sh version (Linux und macOS) in einer Eingabeaufforderung ein. Wenn die Static Analyzer Client Utility-Version, das Verzeichnis 'home' und andere Informationen zurückgegeben werden, sind die Angaben für PATH korrekt.
  4. Gehen Sie wie folgt vor, wenn Sie beabsichtigen, das Plug-in für die statische Analyse zu Eclipse, Microsoft Visual Studio (nur unter Windows), IntelliJ IDEA oder Maven hinzuzufügen: Suchen Sie die Datei install-plugins.bat (Windows) bzw. install-plugins.sh (Linux und macOS) in der extrahierten Instanz von Client Utility und führen Sie die Datei aus. Führen Sie die Datei unter Windows als Administrator aus. Wenn Sie beabsichtigen, das Plug-in zu Eclipse unter Linux oder macOS hinzuzufügen, stellen Sie sicher, dass Sie mit einem Account angemeldet sind, der über Berechtigungen zum Ändern der Eclipse-Instanz verfügt, auf der die Installation ausgeführt wird.
    1. Standardmäßig stellt Client Utility eine Verbindung zum ASoC-Dienst unter IBM Cloud Marketplace her. Wenn die Verbindung zum Dienst stattdessen über IBM Bluemix hergestellt werden soll, geben Sie bei der entsprechenden Eingabeaufforderung yes ein.
    2. Wenn Sie beabsichtigen, Quellcode in Eclipse zu scannen, gehen Sie gemäß den Eingabeaufforderungen im Installationsprogramm vor. Wenn Sie das Eclipse-Plug-in installieren, werden Sie aufgefordert, die Position Ihrer Eclipse-Installation einzugeben. Geben Sie den vollständigen Pfad zu dem Verzeichnis ein, das die Datei eclipse.exe (Windows) oder eclipse (Linux und macOS) enthält.
    3. Wenn Sie vorhaben, Maven zum Generieren von IRX-Dateien zu verwenden, folgen Sie den Eingabeaufforderungen im Installationsprogramm.
    4. Wenn Sie beabsichtigen, Quellcode in IntelliJ zu scannen, gehen Sie gemäß den Eingabeaufforderungen im Installationsprogramm vor. Wenn Sie das IntelliJ IDEA-Plug-in installieren, werden Sie aufgefordert, die Position des Konfigurationsverzeichnisses für IntelliJ IDEA einzugeben. Geben Sie den vollständigen Pfad für dieses Verzeichnis ein.
    5. Nur Windows: Wenn Sie beabsichtigen, Quellcode in Visual Studio zu scannen, gehen Sie gemäß den Eingabeaufforderungen im Installationsprogramm vor. Wenn Sie das Visual Studio-Plug-in installieren, werden Sie aufgefordert, die Version von Visual Studio auszuwählen, für die das Plug-in installiert werden soll. Geben Sie die Version für die Visual Studio-Version an, für die das Plug-in installiert werden soll.
    6. Schließen Sie das Fenster mit der Eingabeaufforderung, indem Sie eine beliebige Taste drücken.
    7. Bei allen integrierten Entwicklungsumgebungen, auf denen Sie das Plug-in installieren, muss nach der Installation ein Neustart ausgeführt werden.
    8. Nur Linux: Wechseln Sie in einer Befehlszeile zu dem Verzeichnis, in dem das Clientdienstprogramm enthalten ist, und fügen Sie die folgenden Zeilen in Ihrer Shellressourcendatei (z. B. .bash_profile oder .shrc) hinzu: 
      export APPSCAN_INSTALL_DIR=<client_utility>
export PATH=${APPSCAN_INSTALL_DIR}/bin:${PATH}
export LD_LIBRARY_PATH=${APPSCAN_INSTALL_DIR}/bin:${LD_LIBRARY_PATH}

      Dabei ist <client_utility> der Pfad zu Client Utility. Laden Sie nach dem Hinzufügen dieser Zeilen Ihre Shell erneut, bevor Sie das Plug-in für die statische Analyse verwenden.

    9. Nur für macOS: Navigieren Sie in einer Befehlszeile zu dem Verzeichnis, in dem das Clientdienstprogramm enthalten ist, und fügen Sie die folgenden Zeilen in Ihrer Shellressourcendatei (z. B. .bash_profile oder .shrc) hinzu: 
      export APPSCAN_INSTALL_DIR=<client_utility>
export PATH=${APPSCAN_INSTALL_DIR}/bin:${PATH}
export DYLD_LIBRARY_PATH=${APPSCAN_INSTALL_DIR}/bin:${DYLD_LIBRARY_PATH}

      Dabei ist <client_utility> der Pfad zu Client Utility. Laden Sie nach dem Hinzufügen dieser Zeilen Ihre Shell erneut, bevor Sie das Plug-in für die statische Analyse verwenden.

      Wichtig: Das Einstellen der Umgebungsvariablen in der Shellressourcendatei betrifft nur Apps, die im Terminal gestartet werden. Sie müssen daher das Terminalfenster zum Starten der IDEs nutzen, über die Sie die statische Analyse ausführen möchten.

Nächste Schritte

Wenn Sie Client Utility auf einem Computer ausführen, der sich hinter einem Proxy befindet: Verwenden Sie eine der folgenden Methoden zur Angabe des Proxys, sodass Client Utility eine Verbindung zur Cloud herstellen kann.

  • Client Utility-CLI und unterstützte IDEs: Setzen Sie diese globale Variable bzw. Systemumgebungsvariable so, dass der Proxy automatisch erkannt wird:
    • Windows: APPSCAN_OPTS=-Dhttps.proxyHost=<Proxy> -Dhttps.proxyPort=<Port>
    • Linux und macOS: APPSCAN_OPTS="-Dhttps.proxyHost=<proxy> -Dhttps.proxyPort=<port>"

    Dabei ist <Proxy> der Hostname des Proxy-Servers und <Port> die Portnummer, die der Proxy-Server verwendet.

    Sie können auch jedes Mal, wenn Sie die Befehlszeilenschnittstelle oder die IDE (die über eine Bedienerführung oder ein Terminal gestartet wird) verwenden, die Client Utility so festlegen, dass der Proxy verwendet wird; hierfür wird ein Befehl wie folgt abgesetzt:

    • Windows: set "APPSCAN_OPTS=-Dhttps.proxyHost=<Proxy> -Dhttps.proxyPort=<Port>"
    • Linux und macOS: export APPSCAN_OPTS="-Dhttps.proxyHost=<proxy> -Dhttps.proxyPort=<port>"
  • Maven: Wenn Sie das Plug-in für die statische Analyse zu Maven hinzugefügt haben, können Sie die Eigenschaft global zu der Umgebungsvariablen MAVEN_OPTS hinzufügen oder die Eigenschaft bei jedem einzelnen Befehl hinzufügen.

Static Analyzer Client Utility entfernen

In dieser Task wird beschrieben, wie die Static Analyzer Client Utility-Plug-ins aus Ihrer integrierten Entwicklungsumgebung (IDE) entfernt werden. Außerdem werden Anweisungen zum Entfernen von Client Utility bereitgestellt. Diese Anweisungen müssen nur dann beachtet werden, wenn Sie vorhaben, die Static Analyzer Client Utility-Komponenten aus Ihrem System zu entfernen. Durch das Durchführen eines Upgrades für diese Komponenten auf eine neue Client Utility-Version werden durch die Plug-in-Installation alte Komponenten automatisch entfernt, bevor die neue Version installiert wird.

Vorbereitende Schritte

Wichtig: Wenn Sie das Client Utility-Plug-in in einer IDE installiert haben, müssen Sie mindestens eine Datei oder einen Befehl an der Position mit der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip ausführen, um das IDE-Plug-in vollständig zu entfernen. Diese Dateien sind folgende:
  • Static Analyzer Client Utility Version 6.0.865 und frühere Versionen:
    • walalib\dotnet.libs35\UnRegisterInGAC.bat
    • walalib\dotnet.libs40\UnRegisterInGAC.bat
  • Alle Static Analyzer Client Utility-Versionen inklusive Version 6.0.865 und frühere Versionen: Öffnen Sie eine Eingabeaufforderung als Administrator und wechseln Sie zur Position, an der Sie die Datei SAClientUtil_<Version>_<Betriebssystem>.zip extrahiert haben. Wechseln Sie in den Ordner bin\ und setzen Sie den folgenden Befehl ab: 
    regsvr32 /u msdia80.dll

Um diese Dateien und Befehle ausführen zu können, löschen Sie den Inhalt der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip erst, wenn alle IDE-Plug-ins entfernt sind.

Sie können diese Dateien und Befehle ausführen, bevor oder nachdem Sie die unten stehenden Anweisungen zum Entfernen der Plug-ins aus Ihrer IDE befolgt haben.

Vorgehensweise

  • Eclipse:
    1. Wählen Sie in der Eclipse-Hauptmenüleiste die Optionen Hilfe > Neue Software installieren aus.
    2. Klicken Sie für die Option Details im Dialogfeld zur Installation auf den Link Was wurde bereits installiert?.
    3. Suchen Sie in der Liste der installierten Software nach Static Analyzer, wählen Sie diese Software aus und wählen Sie anschließend Deinstallieren aus.
    4. Klicken Sie im Dialogfeld für die Deinstallation auf Fertigstellen.
    5. Führen Sie bei Anzeige einer entsprechenden Aufforderung einen Neustart von Eclipse durch.
  • IntelliJ IDEA:
    1. Wählen Sie bei IntelliJ IDEA im Hauptmenü die Optionen Datei > Einstellungen aus.
    2. Wählen Sie im Dialogfeld für die Einstellungen die Option Plug-ins aus.
    3. Suchen Sie in der Liste der Plug-ins nach dem Static Analyzer-Plug-in und wählen Sie es aus.
    4. Klicken Sie auf Deinstallieren.
    5. Klicken Sie auf die Option zum Neustarten von IntelliJ IDEA.
  • Visual Studio:
    1. Wählen Sie bei Visual Studio im Hauptmenü die Optionen Extras > Erweiterungen und Updates aus.
    2. Suchen Sie in der Liste der installierten Erweiterungen und Updates nach IBM Static Analyzer und wählen Sie diese Software aus.
    3. Klicken Sie auf Deinstallieren und klicken Sie anschließend, wenn Sie zur Deinstallation von IBM Static Analyzer aufgefordert werden, auf Ja.
    4. Wählen Sie die Option zum sofortigen Neustart aus, um Visual Studio erneut zu starten, oder wählen Sie Schließen aus, wenn Sie Visual Studio zu einem späteren Zeitpunkt erneut starten möchten.
  • Client Utility: Um Client Utility zu entfernen, müssen Sie nur den Inhalt der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip von Ihrem System entfernen. IRX-Datei über die Befehlszeilenschnittstelle (CLI) generieren

Zur Einleitung einer Analyse Ihrer Dateien müssen Sie eine IRX-Datei generieren, die Sie später an die Cloud übergeben. Wenn Sie die Befehlszeilenschnittstelle verwenden möchten, führen Sie die nachfolgenden Anweisungen zum Erstellen dieser Datei aus.

Vorgehensweise

Gehen Sie wie folgt vor, um die IRX-Datei zu generieren:

  1. Stellen Sie sicher, dass Sie ein neues Fenster mit Eingabeaufforderung geöffnet haben. (Verwenden Sie nicht das Fenster mit Eingabeaufforderung, das zum Installieren von Client Utility verwendet wurde.)
  2. Geben Sie an der Eingabeaufforderung einen Befehl ein, um in das Verzeichnis mit Ihren Dateien zu wechseln. Informationen zu den unterstützten Dateitypen finden Sie Sprachunterstützung. Wechseln Sie in das Verzeichnis des Eclipse-Arbeitsbereichs, wenn Sie Java-Projekte in Eclipse scannen.
    Anmerkung:
    • Dieser Schritt ist nicht erforderlich, wenn Sie eine Konfigurationsdatei zum Generieren der IRX-Datei verwenden. Siehe IRX-Dateigenerierung über die Befehlszeilenschnittstelle konfigurieren.
    • Über die Befehlszeilenschnittstelle können Sie keine einzelnen Projekte in einem Eclipse-Arbeitsbereich, sondern nur den gesamten Arbeitsbereich scannen. Wenn Sie einzelne Projekte scannen wollen, müssen Sie die IRX-Datei in Eclipse generieren.
    • Dateien mit den Erweiterungen .dll und .exe müssen gültige .NET-Assemblys sein.
  3. Setzen Sie den Befehl appscan prepare (Windows) oder den Befehl appscan.sh prepare (Linux und macOS) ab. Dieser Befehl kann mit Optionen verwendet werden, die in den Abschnitten Konfigurationsbefehle (Windows) und Konfigurationsbefehle (Linux und macOS) aufgeführt sind. Sie können zum Beispiel die Optionen -d und -n verwenden, um ein bestimmtes Verzeichnis zum Speichern der IRX-Datei und einen bestimmten Namen für die Datei anzugeben. Darüber hinaus können Sie den Befehl mit einer Konfigurationsdatei ausführen, in der Sie zusätzliche Informationen angeben, die die Generierung einer vollständigen IRX-Datei unterstützen können (siehe IRX-Dateigenerierung über die Befehlszeilenschnittstelle konfigurieren).
    Tipp: Standardmäßig wird Java- und .NET-Code anderer Anbieter beim Generieren der Datei IRX nicht gescannt. Sie können die Ausschlusseinstellungen für Code von Drittanbietern ändern, indem Sie die Anweisungen in Java- und .NET-Ausschlüsse von Drittanbietern verwalten befolgen. Wenn der Code anderer Anbieter einbezogen werden soll, geben Sie die Option -t oder --thirdParty für den Befehl prepare an.

    Wenn Sie Entwickler von Code anderer Anbieter sind, der normalerweise bei einem Scan berücksichtigt würde, sollten Sie die Einstellung zur Einbeziehung von Code anderer Anbieter verwenden.

    Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.
  4. Nach der Eingabe des Befehls sammelt Client Utility Informationen zu allen unterstützten Dateien in dem Verzeichnis, in dem der Befehl eingegeben wurde, sowie in allen zugehörigen Unterverzeichnissen. Eine IRX-Datei wird in diesem Verzeichnis erstellt. Der Name dieser Datei basiert auf dem Verzeichnis, in dem der Befehl ausgeführt wurde, und enthält eine Zeitmarke (Datum/Uhrzeit), sofern Sie nicht über eine Konfigurationsoption einen anderen Dateinamen angegeben haben.
  5. Übergeben Sie die IRX-Datei zur Analyse an die Cloud. Geben Sie dazu den Befehl appscan queue_analysis (Windows) oder den Befehl appscan.sh queue_analysis (Linux und macOS) ein. Dieser Befehl wird mit Optionen verwendet, die in den Abschnitten Analysebefehle (Windows) oder Analysebefehle (Linux und macOS) aufgeführt sind.

Verwendung der Konfigurationsdatei

Eine Konfigurationsdatei kann dazu verwendet werden, die Ziele anzugeben, die bei der Generierung einer IRX-Datei ein- oder ausgeschlossen werden sollen. Darüber hinaus können Sie Verzeichnisse angeben, die erstellte Artefakte enthalten, und Sie können Buildinformationen angeben, die bei der IRX-Dateigenerierung verwendet werden sollen. (Sie können diese Einstellung z. B. zur Angabe eines bestimmten JDK oder eines JSP-Compilers verwenden.)

Wenn Sie den Befehl appscan prepare (Windows) oder den Befehl appscan.sh prepare (Linux und macOS) zum Generieren einer IRX-Datei ausgeben, prüft Static Analyzer Client Utility automatisch, ob sich eine Datei des Typs appscan-config.xml im aktuellen Verzeichnis befindet. Ist die Datei vorhanden, wird sie automatisch verwendet.

Wenn Sie eine Konfigurationsdatei in einem anderen Verzeichnis oder mit einem anderen Namen haben, können Sie die Datei verwenden, indem Sie die Option -c in die Eingabe des Befehls appscan prepare oder des Befehls appscan.sh prepare einschließen. In diesem Fall geben Sie den Befehl appscan prepare -c <Konfigurationsdatei> (unter (Windows)) oder den Befehl appscan.sh prepare -c <konfigurationsdatei> (unter (Linux und macOS)) ein. Dabei ist <Konfigurationsdatei> der vollständige Pfad und der Dateiname der Konfigurationsdatei.

Konfigurationsdateivorlage und Einstellungen

Verwenden Sie zur Vorbereitung der Konfigurationsdatei die folgende Vorlage:

<Configuration>
  <Targets>
    <Target outputs-only="true/false" path="Scanzielpfad">
      <CustomBuildInfo build_info="info"/>
      <Include>string_pattern</Include>
      <Exclude>string_pattern</Exclude>
    </Target>
  </Targets>
</Configuration>

Dabei können Sie mithilfe des Elements Targets die Ziele angeben, die beim Generieren der IRX-Datei gescannt werden sollen. Die Zielattribute (Target) und die untergeordneten Elemente werden wie folgt verwendet:

  • Das Attribut outputs-only des Elements Target wird nur verwendet, wenn Sie ein Verzeichnis scannen und festlegen wollen, dass Client Utility nur die Buildausgabedateien (z. B. Dateien mit den Erweiterungen .jar, .war und .class) suchen soll. Standardmäßig ist dieses Attribut auf false gesetzt. Dies bedeutet, dass Client Utility das Verzeichnis durchsucht, um festzustellen, ob es ein Ziel ist (wie z. B. ein Anwendungsserver oder ein Eclipse-Arbeitsbereich) oder ob das Verzeichnis Elemente wie Build-Scripts, Maven-Dateien (.pom) und Makefiles (make) enthält. Wenn das Scanziel als einfaches Verzeichnis behandelt werden und nur nach Ausgabedateien gesucht werden soll, geben Sie outputs-only="true" im Element Target an.

    Wenn Sie zum Beispiel <Target outputs-only="false" path="C:\Tomcat"> angeben, geht Client Utility davon aus, dass das Ziel Ihr Tomcat-Anwendungsserver ist, und sucht nach den zugehörigen bereitgestellten WAR-Dateien (.war). Wenn Sie jedoch <Target outputs-only="true" path="C:\Tomcat"> angeben, behandelt Client Utility diese Position als Verzeichnis und sucht alle Buildausgabedateien, die sich darin befinden.

  • Verwenden Sie das Attribut path im Element Target, um den Pfad zu einem Scanziel oder einem Verzeichnis mit Scanzielen (<Scanzielpfad>) anzugeben. Wenn ein Verzeichnis angegeben wird, werden alle zugehörigen Unterverzeichnisse in den Scanvorgang einbezogen.

    Beispiel: Wenn Sie <Target outputs-only="false" path="C:\WebSphere\AppServer85\profiles\AppSrv01"> angeben, sucht Client Utility alle im Profil AppSrv01 bereitgestellten EAR-Dateien (.ear). Wenn Sie <Target outputs-only="false" path="C:\WebSphere\AppServer85"> angeben, sucht Client Utility alle implementierten EAR-Dateien (.ear) in allen Profilen.

  • Sie können auch die folgenden optionalen Target-Unterelemente angeben:
    • Mithilfe des Elements CustomBuildInfo können Sie Attribute festlegen. Mit diesem Teil der Vorlage 
      <CustomBuildInfo build_info="info"/>
      können Sie von der Zielsprache abhängige Buildinformationen angeben. Für einige Sprachen können mehrere Attribute festgelegt werden. Beispiel: Sie können abhängig von der Zielsprache <CustomBuildInfo build_info_1="info_1" build_info_2="info_2" build_info_3="info_3"/> festlegen.
      • Für Java können Sie die folgenden Attribute festlegen: 
        <CustomBuildInfo additional_classpath="Abhängigkeitspfad"
  jdk_path="JDK-Pfad" jsp_compiler="JSP-Compilerpfad"/>
        • Geben Sie weitere Klassenpfade mithilfe des Attributs additional_classpath an. Trennen Sie Klassenpfade unter Windows durch ein Semikolon. Trennen Sie mehrere Klassenpfade unter Linux und macOS durch einen Doppelpunkt voneinander.
        • Verwenden Sie das Attribut jdk_path, um den Pfad zu Ihrer JDK-Installation anzugeben.
        • Verwenden Sie das Attribut jsp_compiler, um den Pfad zu Ihrem JSP-Compiler anzugeben. Beispiel: jsp_compiler="C:\Tomcat".
      • Für .NET (nur Windows) können Sie die folgenden Attribute festlegen: 
        <CustomBuildInfo reference="assembly_reference"
  configuration="build_configuration"/>
        • Fügen Sie mithilfe des Attributs reference Assembly-Referenzen hinzu. Trennen Sie Referenzen durch ein Semikolon.
        • Mit dem Attribut configuration können Sie eine Buildkonfiguration für die Erkennung der Lösung Visual Studio einschließen.
      • Für C/C++ (nur Windows) können Sie die folgenden Attribute festlegen: 
        <CustomBuildInfo configuration="build_configuration"
  include_paths="include_directories" macros="macros" compiler_opts=/>
        • Mit dem Attribut configuration können Sie eine Buildkonfiguration einschließen.
        • Geben Sie mithilfe des Attributs include_paths Pfade an. Trennen Sie Include-Pfade durch ein Semikolon.
        • Schließen Sie mithilfe des Attributs macros Makros ein. Trennen Sie Makros durch ein Semikolon.
        • Geben Sie mithilfe des Attributs compiler_opts Compileroptionen an. Trennen Sie Optionen durch ein Semikolon.
    Anmerkung: Die Werte, die durch diese Attribute festgelegt sind, werden von den Unterzielen übernommen. Wenn beispielsweise Ihr Ziel eine EAR-Datei ist, die WAR- und JAR-Unterziele enthält, wird angenommen, dass WAR and JAR dieselben Werte aufweisen, die auch für die EAR-Datei festgelegt sind, die diese Attribute verwendet.
  • Verwenden Sie das untergeordnete Element Include zur Angabe von Dateimustern (<zeichenfolgemuster>), die beim Generieren der IRX-Datei eingeschlossen werden sollen. Das Include-Verhalten hängt vom Zieltyp ab - wie im Abschnitt Include- und Exclude-Verhalten des Ziels beschrieben. Zum Angeben mehrerer Einschlussmuster (include) fügen Sie jedes der Muster in einem eigenen <Include></Include>-Tag hinzu. Beispiel:
    <Include>zeichenfolgemuster_1</Include>
<Include>zeichenfolgemuster_2</Include>
    Anmerkung: Wenn Sie Einschluss- (include) und Ausschlussmuster (exclude) angeben, die in Konflikt zueinander stehen, haben die Ausschlussmuster (exclude) Vorrang.
  • Verwenden Sie das untergeordnete Element Exclude, um Dateimuster anzugeben, die beim Generieren der IRX-Datei ausgeschlossen werden sollen. Das Exclude-Verhalten hängt vom Zieltyp ab - wie im Abschnitt Include- und Exclude-Verhalten des Ziels beschrieben. Zum Angeben mehrerer Ausschlussmuster (exclude) fügen Sie jedes der Muster in einem eigenen <Exclude></Exclude>-Tag hinzu.
    Anmerkung: Wenn Sie Einschluss- (include) und Ausschlussmuster (exclude) angeben, die in Konflikt zueinander stehen, haben die Ausschlussmuster (exclude) Vorrang.

In integrierten Entwicklungsumgebungen scannen

Diese Anweisungen erläutern, wie Sie Quellcode, der bereits auf den integrierten Entwicklungsumgebungen (IDEs) von Eclipse, IntelliJ IDEA oder Visual Studio installiert wurde, mit dem Plug-in zur statischen Analyse scannen. In Eclipse und IntelliJ IDEA können Sie Java-Projekte scannen und in Visual Studio können .NET-Lösungen (C#, ASP.NET, VB.NET) gescannt werden.

Vorbereitende Schritte

Standardmäßig wird Code anderer Anbieter beim Scannen von Java und .NET nicht mit einbezogen. Sie können die Ausschlusseinstellungen für Code von Drittanbietern ändern, indem Sie die Anweisungen in Java- und .NET-Ausschlüsse von Drittanbietern verwalten befolgen.
  • Verwenden Sie eine der folgenden Methoden, um Code anderer Anbieter beim Scannen in Eclipse, IntelliJ IDEA oder Visual Studio einzubeziehen:
    • Setzen Sie vor dem Starten der IDE die folgende globale Variable oder Systemumgebungsvariable: 
      APPSCAN_OPTS=-DthirdParty
    • Bei jeder Verwendung der IDE können Sie vor dem Start der IDE einen Befehl eingeben:
      set APPSCAN_OPTS=-DthirdParty
  • Alternativ können Sie beim Scannen in Eclipse wie folgt vorgehen: Ändern Sie vor dem Start von Eclipse die Datei eclipse.ini so, dass der Abschnitt -vmargs die Angabe -DthirdParty enthält.
  • Alternativ können Sie beim Scannen in IntelliJ IDEA wie folgt vorgehen: Bearbeiten Sie vor dem Start von IntelliJ IDEA im Verzeichnis bin der IntelliJ IDEA-Installation die Datei idea.exe.vmoptions. Fügen Sie -DthirdParty zu der Datei hinzu und speichern Sie sie.

Wenn Sie Entwickler von Code anderer Anbieter sind, der normalerweise bei einem Scan berücksichtigt würde, sollten Sie die Einstellung zur Einbeziehung von Code anderer Anbieter verwenden.

Vorgehensweise

Gehen Sie wie folgt vor, um Quellcode zu scannen und Beurteilungen oder Berichte zu öffnen:

  1. Stellen Sie sicher, dass das Plug-in auf der IDE installiert ist. Wenn die IDE bei der Installation geöffnet war, starten Sie sie erneut.
  2. Wählen Sie das Element aus, das Sie scannen wollen:
    • Wählen Sie in Eclipse das Projekt oder die Projekte aus, das bzw. die Sie scannen wollen. Wählen Sie zum Scannen des gesamten Eclipse-Arbeitsbereichs alle Projekte aus.
    • Wählen Sie in IntelliJ IDEA die Projektmodule aus, die Sie scannen wollen.
    • Wählen Sie in Visual Studio die Lösungen, Projekte oder Websites aus, die Sie scannen wollen.
  3. Klicken Sie mit der rechten Maustaste auf die Auswahl und wählen Sie Sicherheitsanalyse > Statische Analyse ausführen aus.
    Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.
  4. Nach der Scanvorbereitung wird das Dialogfenster für die Anmeldung geöffnet, sofern Sie nicht bereits beim Service angemeldet sind. Welches Dialogfenster geöffnet wird, richtet sich nach der Einstellung für die Verbindung, die nach dem Starten des Installationsprogramms install-plugins.bat (Windows) bzw. install-plugins.sh (Linux und macOS) hergestellt wird. Geben Sie im Dialogfenster Ihre Berechtigungsnachweise für den Service ein:
    • Bei der Herstellung einer Verbindung zum Cloud-Marketplace: Wenn Sie im Application Security on Cloud-Service einen API-Schlüssel generieren, empfangen Sie eine Schlüssel-ID sowie einen geheimen Schlüssel. Geben Sie diese Werte in den Feldern für die ID und den geheimen Schlüssel ein. Haben Sie noch keinen API-Schlüssel generiert, verwenden Sie den Link im Dialogfenster, um einen solchen zu erstellen.
    • Bei der Herstellung einer Verbindung zu Bluemix: Geben Sie für Bindungs-ID und Kennwort die Angaben ein, die in den Serviceberechtigungsnachweisen angegeben sind.

      Die Identifikationsdaten für den Bluemix-Service können Sie ermitteln, indem Sie im Navigationsbereich links im Service-Dashboard Serviceberechtigungsnachweise auswählen. Siehe Externen Apps die Verwendung von Bluemix-Services ermöglichen.

    Bei der Anmeldung am Service wird eine verschlüsselte Schlüsseldatei erstellt. Auf diese Tokendatei wird dann von anderen Aktionen bei der Interaktion mit dem ASoC-Service verwiesen.

    Wenn Sie bei Bluemix angemeldet sind, läuft diese Tokendatei zwei Stunden nach ihrer Erstellung ab. In diesem Fall müssen Sie sich nach diesem Zeitpunkt erneut bei dem Service anmelden, um mit weiteren Aktionen fortfahren zu können. Sie können bei der Anmeldung aber auch das Kontrollkästchen Berechtigungsnachweise speichern auswählen. Bei Auswahl dieses Kontrollkästchens versucht Client Utility automatisch, sich nach Ablauf des Tokens erneut beim Service zu authentifizieren.

  5. Nur IBM Cloud Marketplace: Wenn Sie mit dem Application Security on Cloud-Service im IBM Cloud Marketplace verbunden sind, muss nun für Scans mit der statischen Analyse in Ihrer IDE eine Zuordnung zu einer bereits vorhandenen Application Security on Cloud-Anwendung durchgeführt werden. Nach dem Start des Scans werden Sie in einem Dialogfeld dazu aufgefordert, die Anwendung für die Zuordnung auszuwählen.

    Nach der Auswahl einer Anwendung, der Scans zugeordnet werden sollen, werden die Informationen zu dieser Zuordnung wie folgt gespeichert:

    • Bei Eclipse wird Ihrem gesamten Arbeitsbereich eine Anwendung zugeordnet. Informationen zu der Zuordnung werden in <Arbeitsbereich>/.metadata/.plugins/com.ibm.appscan.eclipse.plugin/.scans/.app gespeichert; dabei ist <Arbeitsbereich> die Position Ihres Eclipse-Arbeitsbereichs.
    • Bei IntelliJ IDEA werden einzelnen Projektmodulen Anwendungen zugeordnet. Informationen zu der Zuordnung werden in <Projekt>/.scans/.app gespeichert; dabei ist <Projekt> die Position Ihres Projektmoduls.
    • Bei Visual Studio werden Lösungen den Anwendungen zugeordnet. Informationen zu der Zuordnung werden in <Lösung>/.scans/.app gespeichert; dabei ist <Lösung> das Verzeichnis, das die Datei mit der Erweiterung .sln enthält.

    Wenn die Datei mit der Erweiterung .app aus einer dieser Positionen entfernt wird, werden Sie beim nächsten Scan zur Auswahl einer zugeordneten Anwendung aufgefordert. Wenn Sie eine Anwendungszuordnung auswählen und zu einer anderen Anwendung wechseln möchten, löschen Sie die Datei mit der Erweiterung .app und wiederholen Sie den Scan.

    Anmerkung: Die Anwendungszuordnung gilt nicht, wenn Sie mit dem Application Security on Cloud-Service unter IBM Bluemix verbunden sind.
  6. Nach der Übergabe des Scans wird die Sicht 'Static Analyzer-Scans' angezeigt.
  7. Wenn der Scan abgeschlossen ist, wird eine Benachrichtigung mit Links geöffnet, über die Sie das Ergebnis bzw. den Bericht öffnen können. Darüber hinaus wird die Sicht 'Static Analyzer-Scans' so aktualisiert, dass sie den Scan enthält. In der Sicht werden der Name des Scans, sein Status, die Start- und Endzeit seiner Ausführung sowie Anzahl und Schweregrad der gefundenen Sicherheitslücken aufgelistet.
    1. Ergebnis: Wählen Sie den Link in der Benachrichtigung aus, um das Ergebnis zu öffnen, oder doppelklicken Sie in der Ergebnis der Sicht auf das entsprechende Symbol. (In Visual Studio kann das Ergebnis auch über die Taskleiste geöffnet werden.) Hierdurch wird eine interaktive Beurteilung mit einer Auflistung der Ergebnisse nach Fixgruppe geöffnet. Eine Fixgruppe stellt den häufigsten Knoten dar, den in Gruppen zusammengefasste Ergebnisse durchlaufen. Durch die Implementierung eines Fixes für eine Fixgruppe kann normalerweise die größte Wirkung bei gleichzeitig geringerem Aufwand erzielt werden. Eine Fixgruppe kann auch als lokaler Punkt für die Gruppierung verstanden werden, an dem zusammengehörige Ergebnisse zur gleichen Zeit überprüft werden können. Beachten Sie, dass eine Fixgruppe nicht gleichbedeutend mit der Position sein muss, an dem ein Fix abgelegt werden sollte. Faktoren wie künftiges Refactoring, Verfahren zur Codeverwaltung usw. können bewirken, dass die Position einer Fixgruppe als Speicherort für Fixes ausgeschlossen ist.

      Folgendes gilt für die Beurteilung:

      • Jede Fixgruppe zeigt die empfohlene Fixposition, einen Link, über den die Quelle dieser Fixposition geöffnet werden kann, die Sicherheitslücke und die Anzahl von Vorkommen der Sicherheitslücke an, die durch die Korrektur des Quellcodes beseitigt werden.
      • Bei Auswahl einer Sicherheitslücke wird die zugehörige Beschreibung geöffnet, die in der Regel Beispiele und Empfehlungen enthält.
      • Wenn Sie die Schaltfläche Details auswählen, werden für die Fixgruppe sämtliche Beurteilungsergebnisse geöffnet, die durch entsprechende Korrektur des Quellcodes angesprochen werden. Wenn Sie in der Detailsicht die Quellenposition auswählen, wird diese im Quelleneditor geöffnet. Bei Auswahl des Tracesymbols wird ein Trace geöffnet, der den Datenfluss durch die Anwendung darstellt.
      Wichtig: Interaktive Beurteilungen (Ergebnisse) sind in IntelliJ IDEA und für bestimmte Subskriptionsebenen nicht verfügbar.
    2. Bericht: Wählen Sie den Link in der Benachrichtigung aus, um den Bericht zu öffnen, oder doppelklicken Sie in der Spalte Bericht der Sicht auf das entsprechende Symbol. (In Visual Studio kann der Bericht auch über die Taskleiste geöffnet werden.) Hierdurch wird ein Bericht mit den Ergebnissen geöffnet. Der Bericht ist auch bei dem Cloud-Service verfügbar, mit dem Sie verbunden sind.

Ergebnisse

Wichtig: Erneutes Scannen wird in integrierten Entwicklungsumgebungen nicht unterstützt.

Beim Generieren einer IRX-Datei (statische Analyse) treten Fehler oder Warnungen auf

Symptome

Bei der Generierung einer IRX-Datei empfangen Sie möglicherweise Fehlernachrichten oder Warnungen oder die Generierung der IRX-Datei schlägt vielleicht fehl. Wenn die Generierung der IRX-Datei fehlschlägt, wird _failed an den Namen der resultierenden Datei angehängt (z. B. my_irx_file_failed.irx). Für derartige Fälle steht eine Vielzahl von Fehlerbehebungsoptionen für die IRX-Dateigenerierung zur Verfügung.

Fehlerbehebung

  • Konfigurationsdatei für die Fehlerbehebung bei der IRX-Dateigenerierung verwenden

    Wenn Sie eine IRX-Datei generieren, ohne hierfür eine Konfigurationsdatei zu verwenden, und die Dateigenerierung nicht abgeschlossen (oder aber mit Fehlern beendet) wird, generiert Static Analyzer Client Utility eine temporäre Konfigurationsdatei, die Sie für nachfolgende Scanvorgänge verwenden können. Diese Datei weist insofern Ähnlichkeit mit der Konfigurationsdateivorlage auf, als dass sie ein Element Targets mit Informationen enthält, die über Ihre Ziele (Targets) bekannt sind. Die Datei enthält jedoch auch das bearbeitbare Element <TargetSettings>, zu dem Sie bei Bedarf Informationen hinzufügen können, durch die Client Utility in die Lage versetzt wird, eine vollständige IRX-Datei zu generieren. Diese Datei hat den Namen appscan-config.xml und wird im aktuellen Verzeichnis gespeichert. Dies bedeutet, dass die Datei automatisch für den nächsten Scanvorgang verwendet wird.

    Beispiel: Bei einer IRX-Dateigenerierung für ein Ziel, das externe Abhängigkeiten enthält, die nicht lokalisiert werden können, empfangen Sie möglicherweise eine Nachricht wie die folgende: 

    C:\myTargetDirectory>appscan prepare
An IRX file was created, but it may be incomplete.
C:\myTargetDirectory\myTarget.jar requires the following packages:
      org.apache.jasper.*
An appscan-config.xml file was created in: C:\myTargetDirectory\appscan-config.xml
Please update the file and rerun the command.

    Die generierte Datei appscan-config.xml entspricht im Wesentlichen dem folgenden Beispiel: 

    <Configuration>
  <Targets>
    <Target outputs-only="true" path="C:\myTargetDirectory"/>
  </Targets>
  <Settings/>
  <TargetSettings>
    <Target path="C:\myTargetDirectory\myTarget.jar"/>
      <!--Please add the path of the following dependencies to the attribute "additional_classpath" below, separated by semi-colon.
			  org.apache.jasper.*-->
      <CustomBuildInfo additional_classpath=""/>
    </Target>
  </TargetSettings>
</Configuration>

    Zum Lösen des Problems aktualisieren Sie <CustomBuildInfo additional_classpath=""/>, um die Klassen org.apache.jasper.* im Klassenpfad einzuschließen. Speichern Sie die Datei appscan-config.xml und führen Sie den Befehl appscan prepare (Windows) oder den Befehl appscan.sh prepare (Linux und macOS) erneut aus. Beachten Sie, dass beim Angeben des Pfads für die JAR-Datei(en) die Standardsyntax für Java-Klassenpfade unterstützt wird (beispielsweise werden Platzhalterzeichen unterstützt).

    Wenn ähnliche Probleme auftreten und Sie bereits eine Konfigurationsdatei verwenden, empfangen Sie nur die Nachricht, die auf die Probleme hinweist. Verwenden Sie diese Nachricht, um Ihre vorhandene Konfigurationsdatei zu aktualisieren, und führen Sie anschließend den Befehl erneut aus.

    Die Konfigurationsdatei ist in IRX-Dateigenerierung über die Befehlszeilenschnittstelle konfigurieren beschrieben.

  • IRX-Dateien im Debugmodus generieren

    Wenn Sie eine IRX-Datei über die Befehlszeilenschnittstelle generieren, können Sie den Befehl appscan prepare (Windows) oder den Befehl appscan.sh prepare (Linux und macOS) mit einer Debugoption absetzen. Dadurch werden weitere Protokolldateien erstellt.

  • Zur IRX-Dateigenerierung gehörige Protokolldateien

    Zum Prüfen von Fehlern oder Warnungen können Sie sich die Protokolldateien ansehen, die ebenfalls generiert werden. Protokolldateien werden in einer .zip-Archivdatei gespeichert. Standardmäßig wird die Datei an der gleichen Position gespeichert, in der auch Ihre IRX-Datei gespeichert ist; Sie können jedoch auch eine andere Position angeben, indem Sie die Option -l des Befehls appscan prepare (Windows) oder des Befehls appscan.sh prepare (Linux und macOS) verwenden. Der Name der .zip-Datei basiert auf dem Namen der IRX-Datei, die generiert wird.

Unterstützte Java-Quellcodeanmerkungen

Wenn die statische Analyse zum Scannen von Java verwendet wird, werden Anmerkungen auf Ebene der Methoden @ValidatorMethod, @CallbackMethod und @SuppressSecurityTrace unterstützt.

  • Anmerkungen verwenden
  • @ValidatorMethod
  • @SuppressSecurityTrace
  • @CallbackMethod

Anmerkungen verwenden

Anmerkungen können verwendet werden, indem Sie die folgenden Schritte beachten:

  1. Die Unterstützung für Anmerkungen ist standardmäßig aktiviert. Die JAR-Datei (.jar) für Anmerkungen befindet sich im Verzeichnis \annotation der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip.
  2. Wenn Sie vorkompilierte Klassen, WAR-Dateien (.war) oder JAR-Dateien (.jar) scanne, fügen Sie com.ibm.security.annotation.jar zu dem Java-Projekt hinzu, das die Quellen mit Anmerkungen enthält, und erstellen Sie das Projekt dann neu.
  3. Wenn Sie den Scan in einer integrierten Entwicklungsumgebung (IDE) vornehmen, in der das Plug-in für die statische Analyse installiert ist, fügen Sie com.ibm.security.annotation.jar zum Java-Klassenpfad hinzu.
  4. Wenn Sie die Befehlszeilenschnittstelle (CLI) für statische Analysen verwenden, müssen Sie zunächst eine Konfigurationsdatei erstellen und in dieser Datei CustomBuildInfo additional_classpath="Abhängigkeitspfad" so festlegen, dass Abhängigkeitspfad die Position der Datei com.ibm.security.annotation.jar enthält.

Anmerkungen können vor dem Scannen zum Quellcode hinzugefügt werden. Sie können aber auch nach dem Scannen und während der Triage hinzugefügt werden, um falsche positive Werte aufzudecken und zu beseitigen. Wenn Sie den Scan in einer integrierten Entwicklungsumgebung (IDE) durchführen, in der das Plug-in für statische Analysen installiert ist, enthält der Scanbericht eine Schaltfläche zur Behebung dieser Situation. Bei Auswahl dieser Schaltfläche werden die Stellen angezeigt, an denen Sie bei Bedarf Anmerkungen hinzufügen können.

Nicht aufgelöste PHP-Ausdrücke include und unbekannte PHP-Klassen konfigurieren

Nach der Generierung einer IRX-Datei für PHP wird eine Konfigurationsdatei erstellt, mit deren Hilfe Sie die IRX-Generierung für das nächste Scannen der Dateien konfigurieren können.

Die Konfigurationsdatei php-scan-config.xml wird an der Position generiert, an der Sie den Befehl prepare ausgegeben haben. Die Datei enthält zwei Abschnitte: <UserInputs> und <Php-Unknown-Configuration>.

Der Abschnitt <UserInputs> enthält Konfigurationseinstellungen, die für Ihren PHP-Scan geändert werden können. Der Abschnitt wird vorab mit Beispielwerten ausgefüllt. Sie können durch Eingabe eines Semikolons (;) Werte an diese Werte anhängen. In diesem Abschnitt sind die folgenden Einstellungen enthalten:

  • <RegularExpression>: Dies ist ein erweitertes Feature, das die Angabe eines regulären Ausdrucks für Suchtext ermöglicht.
  • <ReplaceText>: Diese Einstellung wird für einfache Textsuchen und Textersetzungen verwendet. Der Suchtext wird wie angegeben verwendet und durch den Ersatztext ersetzt, wenn er in dem eingeschlossenen Text gefunden wird.
  • <Function>: Diese Einstellung wird verwendet, wenn ein Funktionsaufruf ersetzt werden soll. Der Suchtext muss der Name der Funktion ohne Klammern sein. Der Suchtext wird erweitert, um nach einem Funktionsaufruf mit dem angegebenen Namen gefolgt von Klammern zu suchen, wobei die Klammern einen beliebigen Inhalt haben können.
  • <SimpleText>: Vereinfachter Ersatztext. Verwenden Sie diese Einstellung, wenn der Ersatztext eine einzelne Zeichenfolge ist.
  • <SimpleRegEx>: Vereinfachter Ersatz für regulären Ausdruck. Verwenden Sie diese Einstellung, wenn der reguläre Ersatzausdruck eine einzelne Zeichenfolge ist.
  • <AdditionalIncludePaths>: Verwenden Sie diese Einstellung, um weitere Include-Pfade (include) hinzuzufügen, die referenzierte Dateien enthalten.

Der Abschnitt <Php-Unknown-Configuration> enthält alle nicht aufgelösten Include-Ausdrücke (include) sowie unbekannte Klassen, die im letzten Scan gefunden wurden. Mit diesen Informationen können Sie den Abschnitt <UserInputs> vervollständigen.

  • <Summary>: Fasst die Anzahl der unbekannten Ausdrücke include (<NumberOfUnknownIncludeExprs>) und die unbekannten Klassen (<NumberOfUnknownClasses>) des letzten Scans zusammen.
  • <SourceFile>: Alle unbekannten Ausdrücke include und unbekannten Klassen werden alphabetisch nach Quellendateiname aufgelistet. Für jede Quellendatei wird im Dateipfad (<FilePath>) der absolute Pfad der Quelle, die unbekannte Klasse (<UnknownClass>) und der unbekannte Ausdruck include (<UnknownIncludes>) aufgeführt.

Informationen zu unbekannten Klassen (<UnknownClass>)

Tag Beschreibung
<ClassName> Der Klassenname, der im Quellcode referenziert wurde, zusammen mit der Position in der Quellendatei (<FilePath>), an der die Klasse verwendet wird. Sie können sich die Position in einem Editor ansehen, um zu prüfen, wie die unbekannte Klasse aufgelöst werden kann.
<Status> Gibt an, ob die Klasse aufgelöst (Resolved) oder nicht aufgelöst (Not Resolved) wurde.
<ResolvedBy> Gibt an, wie die unbekannte Klasse aufgelöst wurde. Mögliche Werte:
  • AutoResolver: Die Anwendung hat interne heuristische Verfahren für die Suche der Datei verwendet.
  • SearchReplace: Eine oder mehrere Regeln zum Suchen und Ersetzen wurden auf den eingeschlossenen Text angewendet, um den aktualisierten Text zu generieren.

Informationen zu unbekannten Ausdrücken include (<UnknownIncludes>)

Tag Beschreibung
<IncludeText> Der Ausdruckstext, wie er im Quellcode enthalten ist, zusammen mit der Position in der Quellendatei (<FilePath>), an der der Ausdruck verwendet wird. Sie können sich die Position in einem Editor ansehen, um zu prüfen, wie der unbekannte include-Ausdruck aufgelöst werden kann.
<UpdatedText> Der aktualisierte Text, der beim letzten Scan verwendet wurde.
<Status> Gibt an, ob der Ausdruck (Resolved) oder nicht aufgelöst (Not Resolved) wurde.
<ResolvedBy> Gibt an, wie der aktualisierte Text generiert wurde. Mögliche Werte:
  • AutoResolver: Die Anwendung hat interne heuristische Verfahren für die Suche der Datei verwendet.
  • SearchReplace: Eine oder mehrere Regeln zum Suchen und Ersetzen wurden auf den eingeschlossenen Text angewendet, um den aktualisierten Text zu generieren.
  • SearchReplace+AutoResolver: Eine oder mehrere Regeln zum Suchen und Ersetzen wurden auf den eingeschlossenen Text angewendet, um den aktualisierten Text zu generieren, und anschließend wurden interne heuristische Verfahren zum Auffinden der Datei angewendet.
  • SearchReplace+IncludePath: Eine oder mehrere Regeln zum Suchen und Ersetzen wurden auf den eingeschlossenen Text angewendet, um den aktualisierten Text zu generieren, und anschließend wurde dieser mit einem Verzeichnis im Include-Pfad (include) kombiniert, um die Datei zu finden.
  • SearchReplace+RelativeDir: Eine oder mehrere Regeln zum Suchen und Ersetzen wurden auf den eingeschlossenen Text angewendet, um den aktualisierten Text zu generieren, der anschließend relativ zum Quellenverzeichnis der Datei, die den Ausdruck include enthält, gefunden wurde.

Java- und .NET-Ausschlüsse von Drittanbietern verwalten

Standardmäßig wird Java- und .NET-Code anderer Anbieter beim Generieren der IRX-Datei nicht gescannt. Sie können den ausgeschlossenen Code von Drittanbietern verwalten, indem Sie die Anweisungen in diesem Abschnitt befolgen.

Informationen zu diesem Vorgang

Bei der Generierung einer IRX-Datei für Ihren Code wird Code von Drittanbietern ausgeschlossen, sodass in Ihrer Beurteilung keine Informationen und Ergebnisse zu Code berücksichtigt werden, den Sie nicht analysieren möchten. Führen Sie die in diesem Abschnitt aufgeführten Schritte aus, um Änderungen in Bezug auf die Ausschlüsse vorzunehmen. Falls Sie jedoch Code von Drittanbietern beim Scannen berücksichtigen möchten, lesen Sie die folgenden Abschnitte:

  • IRX-Datei über die Befehlszeilenschnittstelle (CLI) generieren
  • IRX-Datei für ein Maven-Projekt generieren
  • In integrierten Entwicklungsumgebungen scannen

Vorgehensweise

  1. Suchen Sie das Verzeichnis config der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip (dabei ist <Version> die aktuelle Version von Client Utility).
  2. Suchen Sie in diesem Verzeichnis die Datei dot_net.exclusions (.NET) bzw. die Datei java.exclusions (Java).
    Anmerkung: Die anderen Dateien in diesem Verzeichnis mit der Erweiterung .exclusions sind nur experimentell und eine Änderung dieser Dateien wirkt sich nicht auf Ausschlüsse von Drittanbietern aus.
  3. Öffnen Sie die Datei in einem Texteditor; dort sehen Sie das folgende XML-Format: 
    <?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<Exclusions>
  <packages>
    <package name="package_name"/>
  </packages>
  ...
</Exclusions>
  4. Bearbeiten Sie die Datei, um Pakete für den Ausschluss hinzuzufügen.
    Anmerkung:
    • Angegebene Paketnamen stimmen mit allen Paketen mit demselben Präfix überein (sie werden so behandelt, wie wenn ihnen ein Platzhalterzeichen folgt). Beispiel: Wenn Sie <package name="com.mycompany.common"/> als Ausschluss hinzufügen, werden <package name="com.mycompany.common.action1"/> und <package name="com.mycompany.common.action2"/> ausgeschlossen.
    • Nur Einträge, die zum Container <packages> hinzugefügt werden, wirken sich auf Drittanbieterausschlüsse aus. Das Hinzufügen von Klassen und Methoden wirkt sich nicht auf Ausschlüsse aus.
  5. Speichern Sie die Datei und generieren Sie anschließend eine IRX-Datei für Ihren Quellcode.

Nächste Schritte

Wenn Sie Ausschlussdateien ändern, solten Sie folgende bewährte Verfahren anwenden:
  • Bewahren Sie eine Sicherungskopie der geänderten Ausschlussdateien auf. Beachten Sie insbesondere, dass wenn Sie ein Upgrade auf eine neue Version von Static Analyzer Client Utility durchführen, Sie Versionen von Ausschlussdateien herunterladen, bei denen Ihre Aktualisierungen nicht berücksichtigt sind. Bewahren Sie beim Aktualisieren von Client Utility eine Kopie Ihrer geänderten Ausschlussdateien auf und kopieren Sie sie anschließend über die neue Client Utility-Version.
  • Bewahren Sie Änderungen in einem einzigen Block mit XML-Einträgen auf, damit Sie sie bei Bedarf unkopliziert kopieren und einfügen können.

appscan prepare

Syntax:

appscan prepare -c <Konfigurationsdatei> -d <Speicherpfad> -l <Protokollpfad> -n <Dateiname> -t,--thirdParty -v,--verbose -X,--debug

Beschreibung:

Generiert eine IRX-Datei.

Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.

Optionale Optionen:

  • -c: Diese Option wird zum Konfigurieren eines Scans verwendet.
  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem die IRX-Datei gespeichert werden soll.
  • -l: Geben Sie -l <Protokollpfad> an, wobei <Protokollpfad> das Verzeichnis ist, in dem die Protokolldateien gespeichert werden sollen.
  • -n: Geben Sie -n <dateiname> an, wobei <dateiname> der Dateiname für die IRX ist. Sie können den Dateinamen mit oder ohne die Dateierweiterung .irx angeben. Wenn Sie ihn ohne Erweiterung angeben, wird die Erweiterung automatisch beim Generieren der Datei hinzugefügt.
  • -t,--thirdParty: Standardmäßig wird Java- und .NET-Code anderer Anbieter beim Generieren der Datei IRX nicht gescannt. Sie können die Ausschlusseinstellungen für Code von Drittanbietern ändern, indem Sie die Anweisungen in Java- und .NET-Ausschlüsse von Drittanbietern verwalten befolgen. Wenn der Code anderer Anbieter einbezogen werden soll, geben Sie die Option -t oder --thirdParty für den Befehl prepare an.

    Wenn Sie Entwickler von Code anderer Anbieter sind, der normalerweise bei einem Scan berücksichtigt würde, sollten Sie die Einstellung zur Einbeziehung von Code anderer Anbieter verwenden.

  • -v,--verbose: Geben Sie -v oder --verbose an, um anzugeben, dass eine ausführlichere Informationsausgabe während der IRX-Dateigenerierung angezeigt werden soll.
  • -X,--debug: Geben Sie -X oder --debug an, um den gesamten Befehl im Debugmodus auszuführen. Bei der Ausführung im Debugmodus werden weitere Protokolldateien für die Fehlerbehebung generiert.
Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.

Beispiele:

Zum Generieren einer IRX-Datei, die die Konfigurationsdatei c:\my_config_files\my_config.xml verwendet und die IRX-Datei unter c:\my_irx_files\my_scan.irx speichert, geben Sie den folgenden Befehl ein: 

appscan prepare -c c:\my_config_files\my_config.xml -d c:\my_irx_files -n my_scan.irx

ppscan package

Syntax:

appscan package -d <speicherpfad> -f <beurteilungsdatei> -n <dateiname>

Beschreibung:

Generiert eine IRX-Datei, die eine Beurteilungsdatei (.ozasmt) enthält, die mit einem IBM Security AppScan Source-Produkt der Version 9.0 oder höher erstellt wurde.

Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.

Optionale Optionen:

  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem die IRX-Datei gespeichert werden soll.
  • -f: Geben Sie -f <beurteilungsdatei> an. Dabei ist <beurteilungsdatei> die .ozasmt-Datei, die Sie zum Scannen bereitstellen möchten. Wenn die <Beurteilungsdatei> nicht im aktuellen Verzeichnis enthalten ist, verwenden Sie diese Option, um den Namen und den Pfad der Beurteilungsdatei anzugeben.
    Anmerkung: Diese Option ist nur erforderlich, wenn mindestens eine der folgenden Bedingungen zutrifft:
    • Sie geben den Befehl von einem Verzeichnis aus ein, das mehr als eine Beurteilungsdatei enthält. Wenn das Verzeichnis nur eine Beurteilungsdatei enthält, wird diese Datei verwendet, wenn die Option -f nicht angegeben ist.
    • Sie geben den Befehl von einem Verzeichnis aus ein, das keine Beurteilungsdateien enthält. In diesem Fall muss die Option -f verwendet werden, um den Pfad und den Namen der zu verwendenden Beurteilungsdatei anzugeben.
  • -n: Geben Sie -n <dateiname> an, wobei <dateiname> der Dateiname für die IRX ist. Sie können den Dateinamen mit oder ohne die Dateierweiterung .irx angeben. Wenn Sie ihn ohne Erweiterung angeben, wird die Erweiterung automatisch beim Generieren der Datei hinzugefügt.
Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.

Beispiele:

Im Verzeichnis c:\my_irx_files können Sie eine IRX-Datei erstellen, die diese Beurteilungsdatei enthält (c:\my_assessment_files\my_assessment.ozasmt), indem Sie den folgenden Befehl absetzen: 

appscan package -f c:\my_assessment_files\my_assessment.ozasmt

Die resultierende IRX-Datei wird im Verzeichnis c:\my_irx_files gespeichert.

appscan get_pubkey

Syntax:

appscan get_pubkey -d <Speicherpfad>

Beschreibung:

Lädt den öffentlichen Verschlüsselungsschlüssel zur Verwendung auf einem Computer herunter, der nicht mit dem Internet verbunden ist.

Wenn Sie eine IRX-Datei auf einem Computer generieren, der mit dem Internet verbunden ist, ist dieser Befehl nicht erforderlich, da automatisch ein Verschlüsselungsschlüssel heruntergeladen wird, wenn Sie den Befehl prepare ausgeben. Wenn auf der Maschine bereits ein Verschlüsselungsschlüssel vorhanden ist, wird er bei Bedarf aktualisiert, wenn Sie den Befehl prepare ausgeben.

Wenn Sie jedoch eine IRX-Datei auf einem Computer generieren, der nicht mit dem Internet verbunden ist, können Sie den Verschlüsselungsschlüssel mithilfe dieses Befehls herunterladen. Sie können dann den Verschlüsselungsschlüssel auf den Computer kopieren, der nicht mit dem Internet verbunden ist, damit Sie ihn verwenden können, um die IRX-Datei zu generieren. Um den Verschlüsselungsschlüssel auf diesem Computer verwenden zu können, müssen Sie den Namen der Datei rsa.pub beibehalten und die Datei im Verzeichnis config der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip ablegen (wobei <Version> die aktuelle Version von Client Utility ist).

Anmerkung: Der verwendete Verschlüsselungsschlüssel muss aktuell sein. Wenn eine IRX-Datei mit einem veralteten Verschlüsselungsschlüssel generiert wird, wird die IRX-Datei beim Hochladen abgelehnt. Wenn Ihr Computer mit dem Internet verbunden ist, setzen Sie den Befehl prepare erneut ab, um den Verschlüsselungsschlüssel automatisch zu aktualisieren. Wenn Ihr Computer nicht mit dem Internet verbunden ist, müssen Sie den Befehl get_pubkey verwenden.

Optionale Optionen:

  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem der Verschlüsselungsschlüssel gespeichert werden soll. Wenn diese Option nicht angegeben wird, wird der Schlüssel im Verzeichnis config der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip gespeichert.

appscan.sh status

Syntax:

appscan.sh status -i <Job-ID>

Beschreibung:

Gibt einen der folgenden Statuscodes für einen Analysejob zurück: 

0 = Anstehend (Pending)
1 = Wird gestartet (Starting)
2 = Ist aktiv (Running)
3 = Ausführung beendet (FinishedRunning)
4 = Ausführung mit Fehlern beendet (FinishedRunningWithErrors)
5 = Unterstützung anstehend (PendingSupport)
6 = Bereit (Ready)
7 = Vorbereitung unvollständig (ReadyIncomplete)
8 = Scannen fehlgeschlagen (FailedToScan)
9 = Manuell gestoppt (ManuallyStopped)
10 = Kein (None)
11 = Wird eingeleitet (Initiating)
12 = Konfiguration fehlt (MissingConfiguration)
13 = Konfiguration fehlt möglicherweise (PossibleMissingConfiguration)

Konfigurationsbefehle (Linux und macOS)

Mit Konfigurationsbefehlen können Sie Ihre Dateien zum Scannen vorbereiten.

  • appscan.sh prepare
  • appscan.sh package
  • appscan.sh get_pubkey

appscan.sh prepare

Syntax:

appscan.sh prepare -c <Konfigurationsdatei> -d <Speicherpfad> -l <Protokollpfad> -n <Dateiname> -t,--thirdParty -v,--verbose -X,--debug

Beschreibung:

Generiert eine IRX-Datei.

Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.

Optionale Optionen:

  • -c: Diese Option wird zum Konfigurieren eines Scans verwendet.
  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem die IRX-Datei gespeichert werden soll.
  • -l: Geben Sie -l <Protokollpfad> an, wobei <Protokollpfad> das Verzeichnis ist, in dem die Protokolldateien gespeichert werden sollen.
  • -n: Geben Sie -n <dateiname> an, wobei <dateiname> der Dateiname für die IRX ist. Sie können den Dateinamen mit oder ohne die Dateierweiterung .irx angeben. Wenn Sie ihn ohne Erweiterung angeben, wird die Erweiterung automatisch beim Generieren der Datei hinzugefügt.
  • -t,--thirdParty: Standardmäßig wird Java- und .NET-Code anderer Anbieter beim Generieren der Datei IRX nicht gescannt. Sie können die Ausschlusseinstellungen für Code von Drittanbietern ändern, indem Sie die Anweisungen in Java- und .NET-Ausschlüsse von Drittanbietern verwalten befolgen. Wenn der Code anderer Anbieter einbezogen werden soll, geben Sie die Option -t oder --thirdParty für den Befehl prepare an.

    Wenn Sie Entwickler von Code anderer Anbieter sind, der normalerweise bei einem Scan berücksichtigt würde, sollten Sie die Einstellung zur Einbeziehung von Code anderer Anbieter verwenden.

  • -v,--verbose: Geben Sie -v oder --verbose an, um anzugeben, dass eine ausführlichere Informationsausgabe während der IRX-Dateigenerierung angezeigt werden soll.
  • -X,--debug: Geben Sie -X oder --debug an, um den gesamten Befehl im Debugmodus auszuführen. Bei der Ausführung im Debugmodus werden weitere Protokolldateien für die Fehlerbehebung generiert.
Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.

Beispiele:

Zum Generieren einer IRX-Datei, die die Konfigurationsdatei /root/Desktop/my_config_files/my_config.xml verwendet und die IRX-Datei unter /root/Desktop/my_irx_files -n my_scan.irx speichert, geben Sie den folgenden Befehl ein:

appscan.sh prepare -c /root/Desktop/my_config_files/my_config.xml -d /root/Desktop/my_irx_files -n my_scan.irx

appscan.sh package

Syntax:

appscan.sh package -d <speicherpfad> -f <beurteilungsdatei> -n <dateiname>

Beschreibung:

Generiert eine IRX-Datei, die eine Beurteilungsdatei (.ozasmt) enthält, die mit einem IBM Security AppScan Source-Produkt der Version 9.0 oder höher erstellt wurde.

Anmerkung: Beim Scannen von Quellcode oder beim Generieren einer IRX-Datei wird möglicherweise eine Nachricht angezeigt, in der Sie zur Aktualisierung die aktuelle Version von Client Utility aufgefordert werden. Siehe Kompatibilität von Client Utility mit Serviceversionen in der Cloud.

Optionale Optionen:

  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem die IRX-Datei gespeichert werden soll.
  • -f: Geben Sie -f <beurteilungsdatei> an. Dabei ist <beurteilungsdatei> die .ozasmt-Datei, die Sie zum Scannen bereitstellen möchten. Wenn die <Beurteilungsdatei> nicht im aktuellen Verzeichnis enthalten ist, verwenden Sie diese Option, um den Namen und den Pfad der Beurteilungsdatei anzugeben.
    Anmerkung: Diese Option ist nur erforderlich, wenn mindestens eine der folgenden Bedingungen zutrifft:
    • Sie geben den Befehl von einem Verzeichnis aus ein, das mehr als eine Beurteilungsdatei enthält. Wenn das Verzeichnis nur eine Beurteilungsdatei enthält, wird diese Datei verwendet, wenn die Option -f nicht angegeben ist.
    • Sie geben den Befehl von einem Verzeichnis aus ein, das keine Beurteilungsdateien enthält. In diesem Fall muss die Option -f verwendet werden, um den Pfad und den Namen der zu verwendenden Beurteilungsdatei anzugeben.
  • -n: Geben Sie -n <dateiname> an, wobei <dateiname> der Dateiname für die IRX ist. Sie können den Dateinamen mit oder ohne die Dateierweiterung .irx angeben. Wenn Sie ihn ohne Erweiterung angeben, wird die Erweiterung automatisch beim Generieren der Datei hinzugefügt.
Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.

Beispiele:

Im Verzeichnis /root/Desktop/my_irx_files können Sie eine IRX-Datei erstellen, die diese Beurteilungsdatei enthält (/root/Desktop/my_assessment_files/my_assessment.ozasmt), indem Sie den folgenden Befehl absetzen:

appscan.sh package -f /root/Desktop/my_assessment_files/my_assessment.ozasmt

Die resultierende IRX-Datei wird im Verzeichnis /root/Desktop/my_irx_files gespeichert.

appscan.sh get_pubkey

Syntax:

appscan.sh get_pubkey -d <Speicherpfad>

Beschreibung:

Lädt den öffentlichen Verschlüsselungsschlüssel zur Verwendung auf einem Computer herunter, der nicht mit dem Internet verbunden ist.

Wenn Sie eine IRX-Datei auf einem Computer generieren, der mit dem Internet verbunden ist, ist dieser Befehl nicht erforderlich, da automatisch ein Verschlüsselungsschlüssel heruntergeladen wird, wenn Sie den Befehl prepare ausgeben. Wenn auf der Maschine bereits ein Verschlüsselungsschlüssel vorhanden ist, wird er bei Bedarf aktualisiert, wenn Sie den Befehl prepare ausgeben.

Wenn Sie jedoch eine IRX-Datei auf einem Computer generieren, der nicht mit dem Internet verbunden ist, können Sie den Verschlüsselungsschlüssel mithilfe dieses Befehls herunterladen. Sie können dann den Verschlüsselungsschlüssel auf den Computer kopieren, der nicht mit dem Internet verbunden ist, damit Sie ihn verwenden können, um die IRX-Datei zu generieren. Um den Verschlüsselungsschlüssel auf diesem Computer verwenden zu können, müssen Sie den Namen der Datei rsa.pub beibehalten und die Datei im Verzeichnis config der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip ablegen (wobei <Version> die aktuelle Version von Client Utility ist).

Anmerkung: Der verwendete Verschlüsselungsschlüssel muss aktuell sein. Wenn eine IRX-Datei mit einem veralteten Verschlüsselungsschlüssel generiert wird, wird die IRX-Datei beim Hochladen abgelehnt. Wenn Ihr Computer mit dem Internet verbunden ist, setzen Sie den Befehl prepare erneut ab, um den Verschlüsselungsschlüssel automatisch zu aktualisieren. Wenn Ihr Computer nicht mit dem Internet verbunden ist, müssen Sie den Befehl get_pubkey verwenden.

Optionale Optionen:

  • -d: Geben Sie -d <Speicherpfad> an, wobei <Speicherpfad> das Verzeichnis ist, in dem der Verschlüsselungsschlüssel gespeichert werden soll. Wenn diese Option nicht angegeben wird, wird der Schlüssel im Verzeichnis config der extrahierten Datei SAClientUtil_<Version>_<Betriebssystem>.zip gespeichert.

appscan.sh api_login

Syntax:

appscan.sh api_login -P <geheimer Schlüssel> -u <Schlüssel-ID> -persist

Beschreibung:

Dient zur Anmeldung beim ASoC-Service in IBM Cloud Marketplace mithilfe eines generierten API-Schlüssels.

Wichtig: Der Befehl api_login wird nur zum Anmelden beim ASoC-Service in IBM Cloud Marketplace über die Befehlszeilenschnittstelle verwendet. Verwenden Sie den Befehl login, um sich beim ASoC-Service in Bluemix anzumelden.

Wenn Sie sich beim Service anmelden, wird eine verschlüsselte Schlüsseldatei mit dem Namen cli.token im Ordner .appscan Ihres Verzeichnisses home gespeichert. Auf diese Tokendatei wird dann von anderen Befehlen und Aktionen bei der Interaktion mit dem ASoC-Service verwiesen. Diese Tokendatei läuft zwei Stunden nach ihrer Erstellung ab. Nach diesem Zeitpunkt müssen Sie sich erneut bei dem Service anmelden, um mit weiteren Befehlen oder Aktionen fortfahren zu können. Oder Sie geben bei der Anmeldung die Option -persist an. Bei Angabe dieser Option versucht Client Utility automatisch, sich nach Ablauf des Tokens erneut für den Service zu authentifizieren.

Erforderliche Optionen:

  • -P: Geben Sie -P <geheimer Schlüssel> an; dabei ist <geheimer Schlüssel> der geheime Schlüssel, der erstellt wurde, als Sie einen API-Schlüssel generiert haben.
  • -u: Geben Sie -u <Schlüssel-ID> an; dabei ist <Schlüssel-ID> die Schlüssel-ID, die erstellt wurde, als Sie einen API-Schlüssel generiert haben.

Optionale Optionen:

  • -persist: Nach Ablauf der Tokendatei für die Anmeldung wird automatisch versucht, sich erneut für den Service zu authentifizieren.
Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.

Ergebnisbefehle (Linux und macOS)

Mit Ergebnisbefehlen können Sie Scanergebnisse aus dem Analyseservice abrufen.

appscan.sh get_result

Syntax:

appscan.sh get_result -d <dateipfad> -i <Job-ID> -t <Typ>

Beschreibung:

Ruft Scanergebnisse ab.

Stellen Sie vor Verwendung dieser Befehle sicher, dass Sie sich beim Analyseservice angemeldet haben.

Erforderliche Optionen:

  • -i: Geben Sie -i <Job-ID> an, wobei <Job-ID> die ID des Analysejobs ist.

Optionale Optionen:

  • -d: Geben Sie -d <dateipfad> an, wobei <dateipfad> der vollständig qualifizierte Pfad für die Zieldatei und/oder der Dateiname der Zieldatei ist. Wenn kein Dateiname angegeben wird, wird der Dateiname auf der Basis des Scan-Jobnamens generiert. Wenn kein Pfad angegeben wird, wird die Datei im aktuellen Verzeichnis gespeichert. Wenn diese Option nicht angegeben wird, wird die Datei im aktuellen Verzeichnis unter einem Dateinamen gespeichert, der auf dem Scan-Jobnamen basiert.
  • -t: Geben Sie -t <Typ> an. Dabei ist <Typ> entweder html oder zip. Die Ergebnisse werden als HTML-Datei oder als .zip-Datei, die die HTML-Ergebnisse enthält, gespeichert. Wenn diese Option nicht angegeben ist, werden die Ergebnisse als HTML-Datei gespeichert.

    Wenn die Scanergebnisse für eine IRX-Datei erstellt werden, die mit dem Befehl package generiert wurde, können Sie durch Angeben der Option -t zip Ergebnisse speichern, die eine neue .ozasmt-Datei enthalten, die in Ihr AppScan Source-Produkt der Version 9.0 oder höher geladen werden kann.

Tipp: Bei allen Befehlen können Optionen in beliebiger Reihenfolge angegeben werden.
